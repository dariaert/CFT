# Утилита фильтрации содержимого файлов

---

## Пример запуска

```bash
java -jar target/CFT-1.0-SNAPSHOT.jar -s -p sample- in1.txt in2.txt
```

---

## Инструкция по запуску
1. Версия Java: `25.0.2`
2. Сборка: `maven 3.9.11`

---

## Формат входных данных

- Входные файлы - текстовые (`.txt`)
- Каждое значение располагается на отдельной строке
- Тип данных определяется автоматически
  - Целые числа (INTEGER)
  - Вещественные числа (FLOAT)
  - Строки (STRING)
- Пустые строки игнорируются

### Пример:
```
Lorem ipsum dolor sit amet
45
Пример
3.1415
consectetur adipiscing
-0.001
тестовое задание
100500
```

## Формат выходных данных

По умолчанию в текущей директории создаются следующие файлы:

- `integers.txt` - целые числа
- `floats.txt` - вещественные числа
- `strings.txt` - строки

Файлы создаются **только при наличии данных соответствующего типа**.

### Настройка имен и расположения файлов

- Опция `-o <path>` позволяет указать путь для выходных файлов
- Опция `-p <prefix>` добавляет префикс к именам файлов
- Опция `-a` позволяет добавлять данные в существующие файлы вместо перезаписи

#### Пример:
```bash
java -jar target/CFT-1.0-SNAPSHOT.jar -f -a -o output -p result_ in1.txt in2.txt
```

#### Создаст файлы:
```
output/result_integers.txt
output/result_floats.txt
output/result_strings.txt
```

---

## Аргументы командной строки

| Опция | Описание |
|------|----------|
| `-s` | вывод краткой статистики |
| `-f` | вывод полной статистики |
| `-a` | режим добавления в существующие файлы |
| `-o <path>` | путь для выходных файлов |
| `-p <prefix>` | префикс имён выходных файлов |

## Обработка ошибок

- Если входной файл не существует - выводится сообщение об ошибке, программа продолжает работу с другими файлами
- Если опция требует аргумент (`-o` или `-p`), а аргумент не указан - выводится сообщение об ошибке
- Пустые строки игнорируются
- При невозможности создать или записать файл - выводится ошибка, программа продолжает обработку других данных
